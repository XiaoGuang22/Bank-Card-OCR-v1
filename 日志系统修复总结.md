## 修复概述
本次修复主要解决了操作日志面板（AuditLogPanel）和相机状态栏（CameraStatusBar）在用户退出登录时出现的资源竞争和UI访问异常问题。

## 问题一：退出登录时日志面板崩溃

### 问题描述
用户退出登录时，系统出现以下异常：
```
RuntimeError: main thread is not in main loop
TclError: invalid command name ".!frame2.!frame3"
```

### 问题原因分析

#### 根本原因
**资源竞争和生命周期管理问题**：退出登录时存在以下执行顺序问题：

1. **记录退出日志** → 触发 `_audit()` 方法
2. `_audit()` 调用 `audit_log_panel.append_log()` 
3. `append_log()` 写入数据库后，调用 `self.frame.after(0, self._load_recent)` 安排UI刷新
4. **销毁日志面板** → `audit_log_panel.destroy()` 销毁UI控件
5. **后台线程完成查询** → 尝试通过 `frame.after()` 更新已销毁的UI

#### 技术细节
- `append_log()` 在后台线程中执行数据库查询
- 查询完成后通过 `frame.after()` 回到主线程更新UI
- 但此时 `frame` 已被 `destroy()` 销毁
- 导致 `TclError: invalid command name` 异常

#### 时序图
```
主线程                          后台线程
  |                               |
  |-- _audit() ------------------>|
  |                               |-- 写入数据库
  |                               |-- 启动查询线程
  |-- destroy() 销毁面板          |
  |   (frame 已销毁)              |
  |                               |-- 查询完成
  |                               |-- frame.after() ❌ 崩溃
  |                               |   (访问已销毁的控件)
```

### 解决方案

#### 方案一：调整销毁顺序（已实施）
**在 `InspectMainWindow.logout()` 中调整执行顺序**：

```python
def logout(self):
    # ★★★ 步骤1: 先销毁日志面板 ★★★
    if hasattr(self, 'audit_log_panel') and self.audit_log_panel:
        self.audit_log_panel.destroy()
        self.audit_log_panel = None  # 清空引用
    
    # ★★★ 步骤2: 再记录退出日志 ★★★
    # 此时日志面板已销毁，不会触发UI刷新
    self._audit("login", "logout")
    
    # 步骤3: 其他清理工作...
```

**优点**：
- 简单直接，修改最小
- 确保日志面板销毁后不会再被访问

**缺点**：
- 退出日志仍会写入数据库，但不会在UI显示
- 如果 `_audit()` 在其他地方被调用，仍可能出问题

#### 方案二：添加销毁标志（已实施，推荐）
**在 `AuditLogPanel` 中添加 `_destroyed` 标志**：

```python
class AuditLogPanel:
    def __init__(self, parent, viewer_name, viewer_role):
        self._destroyed = False  # ★★★ 添加销毁标志 ★★★
        # ...
    
    def _load_recent(self):
        """加载日志前检查是否已销毁"""
        if self._destroyed:  # ★★★ 检查销毁标志 ★★★
            return
        # ...
    
    def _fetch_and_render(self, keyword: str):
        """后台查询前检查是否已销毁"""
        if self._destroyed:  # ★★★ 检查销毁标志 ★★★
            return
        # ...
        
        # 回到主线程前再次检查
        if not self._destroyed and self.frame.winfo_exists():
            self.frame.after(0, lambda: self._render_rows(rows))
    
    def _render_rows(self, rows: list):
        """渲染前检查是否已销毁"""
        if self._destroyed:  # ★★★ 检查销毁标志 ★★★
            return
        # ...
    
    def append_log(self, ...):
        """追加日志前检查是否已销毁"""
        if self._destroyed:  # ★★★ 检查销毁标志 ★★★
            return
        
        # 写入数据库...
        
        # 刷新UI前再次检查
        if not self._destroyed:
            try:
                self.frame.after(0, self._load_recent)
            except tk.TclError:
                pass  # 窗口已销毁，忽略异常
    
    def destroy(self):
        """销毁时设置标志"""
        self._destroyed = True  # ★★★ 设置销毁标志 ★★★
        try:
            self.frame.destroy()
        except:
            pass
```

**优点**：
- 彻底解决资源竞争问题
- 所有后台线程和回调都会检查销毁状态
- 防御性编程，更加健壮
- 适用于任何调用场景

**缺点**：
- 需要在多个方法中添加检查代码

#### 方案三：使用 `winfo_exists()` 检查（辅助方案）
在访问UI控件前检查其是否存在：

```python
try:
    if self.frame.winfo_exists():
        self.frame.after(0, callback)
except tk.TclError:
    pass  # 控件已销毁，忽略
```

**优点**：
- 直接检查控件状态
- 配合销毁标志使用更安全

### 最终实施方案
**组合方案一 + 方案二 + 方案三**：

1. **调整销毁顺序**：先销毁日志面板，再记录退出日志
2. **添加销毁标志**：在所有可能的异步访问点检查 `_destroyed`
3. **异常捕获**：使用 `try-except` 捕获 `TclError`

## 问题三：日志表格显示优化

### 问题描述
操作日志中的"目标对象"列经常显示很长的文本（如相机切换信息），导致：
- 表格列宽不够，文本被截断
- 用户需要横向滚动才能看到完整内容
- 影响用户体验

### 解决方案

#### 1. 增加行高支持多行显示
```python
# 创建自定义样式，增加行高以支持多行显示
style = ttk.Style()
style.configure("Multiline.Treeview", rowheight=40)  # 设置行高为40像素（约2行文本）

self._tree = ttk.Treeview(
    table_frame, columns=cols, show="headings",
    selectmode="browse", height=6,
    style="Multiline.Treeview"  # 应用自定义样式
)
```

#### 2. 自动换行处理
```python
def _wrap_text(self, text: str, max_width: int = 30) -> str:
    """
    将长文本按指定宽度自动换行
    
    Args:
        text: 原始文本
        max_width: 每行最大字符数
        
    Returns:
        换行后的文本
    """
    if not text or len(text) <= max_width:
        return text
    
    # 按箭头符号分割（相机切换格式：A → B）
    if " → " in text:
        parts = text.split(" → ")
        if len(parts) == 2:
            return f"{parts[0]}\n→ {parts[1]}"
    
    # 普通文本：每max_width个字符换行
    lines = []
    for i in range(0, len(text), max_width):
        lines.append(text[i:i+max_width])
    return "\n".join(lines)
```

#### 3. 应用换行到目标对象列
```python
def _render_rows(self, rows: list):
    for i, r in enumerate(rows):
        # 对目标对象进行自动换行处理（每30个字符换行）
        target_obj = r["target_object"]
        target_obj_wrapped = self._wrap_text(target_obj, max_width=30)
        
        self._tree.insert(
            "", tk.END,
            values=(
                ts_str,
                r["user_name"],
                r["user_role"],
                op_type,
                op_action,
                target_obj_wrapped,  # 使用换行后的文本
                result,
                r["ip_address"],
            ),
            tags=(tag_result, tag_row),
        )
```

#### 4. 列居中对齐
```python
for col in cols:
    self._tree.heading(col, text=col)
    self._tree.column(col, width=col_widths.get(col, 80),
                      minwidth=40, anchor="center")  # 改为居中对齐
```

### 效果
- 长文本自动换行显示
- 相机切换信息格式化为两行（`相机A\n→ 相机B`）
- 行高增加到40像素，可容纳2行文本
- 所有列居中对齐，视觉效果更好

## 技术要点总结

### 1. Tkinter 多线程编程最佳实践
- **原则**：只在主线程中操作UI控件
- **方法**：使用 `widget.after()` 将UI操作调度到主线程
- **检查**：在回调前检查控件是否仍然存在

### 2. 资源生命周期管理
- **销毁标志**：使用布尔标志跟踪对象销毁状态
- **防御性编程**：在所有异步访问点检查销毁标志
- **异常处理**：捕获 `TclError` 等UI相关异常

### 3. 后台线程与UI交互模式
```python
def async_operation(self):
    """异步操作模板"""
    # 1. 检查销毁标志
    if self._destroyed:
        return
    
    # 2. 执行后台任务
    result = do_background_work()
    
    # 3. 回到主线程更新UI
    if not self._destroyed:
        try:
            if self.widget.winfo_exists():
                self.widget.after(0, lambda: self._update_ui(result))
        except tk.TclError:
            pass  # 控件已销毁，忽略
```

### 4. 组件销毁顺序
```python
def cleanup(self):
    """正确的清理顺序"""
    # 1. 设置销毁标志（阻止新的异步操作）
    self._destroyed = True
    
    # 2. 停止定时器和后台任务
    self._stop_timers()
    
    # 3. 取消注册回调
    self._unregister_callbacks()
    
    # 4. 销毁UI控件
    try:
        self.widget.destroy()
    except:
        pass
```

### 2. 封装通用基类

```python
class SafeFrame(tk.Frame):
    """带销毁保护的Frame基类"""
    
    def __init__(self, parent, **kwargs):
        super().__init__(parent, **kwargs)
        self._destroyed = False
    
    def safe_after(self, ms, callback):
        """安全的after调用"""
        if not self._destroyed:
            try:
                if self.winfo_exists():
                    self.after(ms, lambda: self._safe_callback(callback))
            except tk.TclError:
                pass
    
    def _safe_callback(self, callback):
        """安全的回调执行"""
        if not self._destroyed:
            try:
                callback()
            except Exception as e:
                print(f"回调异常: {e}")
    
    def destroy(self):
        """销毁时设置标志"""
        self._destroyed = True
        super().destroy()
```

## 总结

本次修复通过以下三个关键措施彻底解决了日志系统的资源竞争问题：
