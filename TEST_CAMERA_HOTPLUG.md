# 相机热插拔功能测试指南

## 快速测试步骤

### 1. 启动程序

```bash
cd d:\CardProject7.0\Bank-Card-OCR-v1
python main.py
```

### 2. 检查初始化

在控制台中查找以下输出：

```
✅ [Sapera] ServerNotify 事件已注册（用于日志记录）
✅ [Sapera] 同时启动轮询检测作为备用方案（5秒间隔）
✅ [InspectMainWindow] ServerNotify 日志记录器已注册
```

如果看到这三行，说明功能已正确初始化。

### 3. 测试相机离线

#### 操作步骤
1. **拔掉相机网线**（或关闭相机电源）
2. **等待 5-10 秒**（轮询间隔）
3. **观察控制台输出**

#### 预期输出

```
[Sapera] 轮询检测: 相机离线 - Genie_M1600_1
[InspectMainWindow] 相机离线日志已记录: S1049704 (192.168.11.136)
```

#### 验证日志

1. 点击主界面的**"操作日志"**标签页
2. 查看最新的日志记录
3. 应该看到类似以下内容：

| 时间 | 用户 | 角色 | 操作类型 | 具体动作 | 目标对象 | 结果 |
|------|------|------|----------|----------|----------|------|
| 2026-05-18 14:30:15 | admin | 管理员 | 相机设置 | **相机离线** | S1049704 (192.168.11.136)<br>(Genie_M1600_1) | 成功 |

- ❌ **不应该**自动刷新相机下拉框
- ❌ **不应该**自动切换相机

## 常见问题排查

### 问题 1：没有看到轮询启动日志

**症状**：
```
❌ 没有看到 "[Sapera] 同时启动轮询检测作为备用方案（5秒间隔）"
```

**可能原因**：
1. Sapera SDK 未正确加载
2. 代码修改未生效

**解决方法**：
```bash
# 重新编译 Python 文件
python -m py_compile camera\sapera_camera_discovery.py

# 重启程序
python main.py
```

---

### 问题 2：插拔相机后没有日志输出

**症状**：
```
❌ 拔掉相机网线后，等待 10 秒仍然没有任何输出
```

**可能原因**：
1. 轮询未启动
2. 相机未被 Sapera SDK 识别为服务器

**排查步骤**：

1. **检查轮询是否启动**：
   ```
   # 应该看到这行日志
   [Sapera] 同时启动轮询检测作为备用方案（5秒间隔）
   ```

2. **手动触发扫描**：
   - 点击相机状态栏的"刷新"按钮
   - 观察控制台输出的服务器列表

3. **检查服务器名称**：
   ```
   [Sapera] Found 3 server(s)
   [Sapera] Server 0: System
   [Sapera] Server 1: Genie_M1600_1  ← 应该看到相机服务器
   [Sapera] Server 2: Genie_M1600_2
   ```

---

### 问题 3：日志记录了但操作日志面板没有显示

**症状**：
```
✅ 控制台输出：[InspectMainWindow] 相机离线日志已记录
❌ 操作日志面板中看不到记录
```

**可能原因**：
1. 日志面板未刷新
2. 权限过滤（操作员只能看到自己的日志）

**解决方法**：

1. **手动刷新日志面板**：
   - 点击操作日志面板的"刷新"按钮

2. **检查用户权限**：
   - 管理员：可以看到所有日志
   - 技术员：可以看到自己和操作员的日志
   - 操作员：只能看到自己的日志

3. **检查数据库**：
   ```bash
   # 查看日志数据库
   sqlite3 Logs\audit_log.db
   
   # 查询最新的 10 条日志
   SELECT * FROM audit_logs ORDER BY timestamp DESC LIMIT 10;
   ```

---

### 问题 4：轮询间隔太长

**症状**：
```
⚠️ 拔掉相机后需要等待 10 秒才看到日志
```

**说明**：
- 这是正常现象
- 轮询间隔为 5 秒，最坏情况下需要等待 5 秒
- 如果刚好在两次轮询之间拔掉相机，可能需要等待接近 10 秒

**优化方法**（可选）：

修改 `camera/sapera_camera_discovery.py`：

```python
# 将轮询间隔从 5 秒改为 2 秒
self._polling_interval = 2.0  # 2秒检测一次
```

**注意**：间隔越短，CPU 占用越高（但仍然很低）
